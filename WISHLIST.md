# Вишлист доработок

Сюда можно накидывать идеи улучшений и доработок. По мере реализации отмечайте галочкой, переносите в архив или удаляйте.

## Как вести

- Задачи оформляются как чекбоксы: `- [ ]` — не сделано, `- [x]` — сделано (можно править вручную в любом редакторе или на GitHub).
- Кратко формулируйте идею в одной строке или небольшом абзаце после чекбокса.
- По желанию добавляйте теги: `[идея]`, `[важно]`, `[позже]`, `[исследовать]` — в тексте пункта.
- Дата в архиве — по желанию.

## Идеи

- [ ] [логирование / аудит] Логирование входов на клиентский портал и в админку (успех, неудача, при необходимости IP, время, идентификатор пользователя).
- [ ] [админка / UX] Временное отключение клиентского портала из админки с показом пользователям сообщения о техническом обслуживании.
- [ ] [производительность] Кеширование ЭЦП при пробросе смарт-карт на время сессии для ускорения работы.
- [ ] [инфраструктура] Резервное копирование (данных/конфигурации — уточнить объём при реализации).
- [ ] [интеграция / AD] Автоимпорт серверов из Active Directory: периодическая синхронизация по LDAP, при необходимости фильтры (OU, группы, атрибуты).
- [ ] [HA] Настройка нод и режима работы высокой доступности (HA) через админку.
- [ ] [админка] Система автообновления (проверка/установка обновлений) из админки.
- [ ] [i18n] Вынести все языковые блоки (надписи UI, уведомления, сообщения об ошибках) в отдельную структуру — ключи и файлы локализации (например `locales/` или аналог) — для дальнейшей реализации мультиязычности.
- [ ] [протокол] Реализация транспорта RDP по UDP (RDPEUDP / UDP для мультимедиа и улучшенной работы при нестабильной сети).
